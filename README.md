# Welcome to Your First GitHub Project! 🎉

You made it — this is your very first step into the world of coding and collaboration. This repository (or "repo" for short) is a hands-on tutorial that will walk you through everything you need to get started with GitHub, VS Code, and deploying your own website/web-app.

By the end, you'll have a live site/app on the internet with a link you can share. For real.

This guide is for students and for anyone else I share it with: friends, colleagues, and administrators who want a practical path from starter code to a live project.

![GitHub Pages Deploy](https://img.shields.io/badge/GitHub%20Pages-Deploy-blue?logo=github)

---

## What You'll Learn

- **GitHub**: account setup, repositories, and workflow basics
- **Codespaces**: cloud development with no local install required
- **VS Code**: editing, extensions, and source control workflows
- **Git**: commit history and local repo fundamentals
- **GitHub Pages**: free website/app hosting from your repository

---

## Step 0: Start Here First (Required)

Before you begin Step 1, you must be in your **own repository copy**.

1. Create a GitHub account (if needed).
2. Open this starter repository.
3. Click **Use this template**.
4. Create a new repository under your own account.
5. Continue the tutorial in your new repository.

> Progress checkboxes and issue closing should happen in your own repo, not the original starter repo.

---

## Choose Your Tracking Mode

This starter supports two modes so you can test both experiences.

### Track A (Recommended): File-Based Steps + Progress Tracker Issue

- Read each step file in `docs/steps/`.
- Create one issue from the **Tutorial Progress Tracker** issue template.
- Check off progress in that single tracker issue.

### Track B (Optional): Generated Step Issues

- Open the **Actions** tab in your own repo.
- Run the **Initialize Tutorial Issues** workflow.
- It will create one issue per step from `docs/steps/`.
- Complete each issue and close it as you go.
